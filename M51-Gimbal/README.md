# M51
RMUC-2025赛季英雄出现的问题
1.底盘超限：（偶发性超限 概率较低 但场上稳定可以稳定复现一次两次）
  超限现象：ui界面60J缓冲能量快速清零
  最终观测ozone曲线发现端倪：在解包裁判系统PowerLimit限制值时发现曲线竟然会进行跳变，最终定位到解包裁判系统代码出现问题，我们使用串口dma-circlr模式进行接收，然后单片机进行处理，发现修改初始化的缓冲区为128 Byte大小会基本解决这个问题。
2.发射机构不稳定：
  结构不稳定性总结：结构链路上的拨弹盘供弹链路摩擦轮均出现了问题，拨弹盘正常拨由于下方为打印件不是轴承阻力会稍大，供弹链路的特氟龙管被蹭出来导致拨弹延迟与双发问题，双级摩擦轮的卡簧预置结构并不能稳定发挥预置作用，总结下这些拨弹链路卡簧结构的细节问题导致发射机构功能出现严重不稳定性，甚至出现卡第一级摩擦轮打不出弹情况！！！
  嵌入式：拨弹盘电机为减速比为50左右的3508电机，每拨一次弹链路往前推进60度，但拨弹时出现链路长度变化问题，观测ozone曲线发现连续拨30发左右，误差只有两度以内。我们采用PID角度闭环控制，累积拨弹角度（rad），并使用FSM（状态机）处理拨弹盘卡弹情况：策略为如果检测堵转电流大于一定阈值且超过一定时间（150ms左右）就进行回退20度然后等待回拨后（耗费100ms）后再进行前推20度，以此保持链路长度不变。但是实际效果并未如此，拨弹误差会大的多，卡簧预置出现被拉开情况，我们最终定位到减速比过大可能会导致软件计算圈数出现丢圈问题，但并未解决。
3.云台控制的问题
  Yaw轴采用6020电机并使用2：1减速比同步带结构，这导致在比赛中云台掉电会存在云台正方向反向问题。
  Pitch轴采用2006电机并使用丝杠结构控制，但是丝杠依旧存在发射弹丸瞬间会存在抖动现象，除此之外pitch轴响应很慢在比赛中并不能灵活的应对中远距离吊射时快速调整以及操作流畅性需求
    针对于IMU零飘导致的影响发射机构散布问题，我们最终采用编码器角度闭环控制但是存在此模式下yaw轴电机与底盘耦合在一起底盘必须失能问题，这也导致操作复杂了很多，以至于在RMUC-2025区域赛中出现操作手失误问题。
  IMU零飘问题现在十分严峻，严重影响发射机构的功能。
4.图传与倍镜控制
  比赛中出现过两次pitch轴图传电机使用键鼠控制不了问题，但是键鼠逻辑并未存在问题其他键鼠功能也可以正常跑，我深感疑惑，不明白的时roll轴倍镜与pitch轴图传控制逻辑一样的前提下，pitch轴控制会出现问题，这个问题我还未解决
  图传pitch轴调整是用鼠标滚轮控制，虽然已经做了一键切换角度，但是操作手在比赛中需要调整图传pitch轴角度，这个非常消耗时间且UI画线也变得毫无参考意义，如果没有自瞄辅助的话需要预先试错好多发弹丸才可以，及其消耗时间与经济，且摩擦轮开启后图传画面在共振的影响下也是微微晃动，这也导致了操作手在进行吊射弹道与落点预测上唇线一些问题。
